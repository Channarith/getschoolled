HARVESTER RUNBOOK - 24/7 large-scale course-material ingestion (100k+)
======================================================================

Runs on a SEPARATE long-lived worker agent (its own thread/pod), independent of
the app stack. It only ingests permissively-licensed / OER material: the license
gate (aoep_shared.harvest.is_allowed) runs BEFORE any fetch.

PIPELINE (aoep_shared/harvest)
  seeds -> HarvestQueue (URL dedup) -> HarvestWorker.process per source:
    license gate -> fetch (robots + rate-limit) -> extract -> content dedup ->
    normalize to a Deck -> POST to curriculum /decks + register in the catalog.

WIRING THE REAL WORKER (on the worker agent)
  - fetcher:   httpx GET honoring robots.txt + a polite rate limit + retries.
  - extractor: services/curriculum/.../ingest.py (extract_pdf/html/transcript ->
               sections_to_deck) keyed off SourceSpec.source_type.
  - sink:      POST {CURRICULUM_URL}/decks and add a catalog Course/Module.
  Then: HarvestWorker(fetcher, extractor, sink).run(queue)  in a loop.

REQUIRED SECRETS / ENV (Cursor Secrets panel or the worker agent env)
  - CURRICULUM_URL        curriculum service base URL to post decks/catalog
  - OBJECT_STORE_*        S3/MinIO creds for storing raw artifacts (see cloud.env)
  - HARVEST_USER_AGENT    contact UA string for polite crawling
  - HARVEST_MAX_RPS       global rate limit (politeness)
  - HARVEST_SEEDS         path/URL of the seed list (OER portals, OCW, etc.)
  - (optional) source API keys if pulling from APIs instead of crawling

SCALE / DURABILITY (production swaps for the in-memory stubs)
  - Back HarvestQueue with Redis/DB so progress survives restarts; shard by host.
  - Dedup with persistent URL + content (minhash) stores.
  - Batch + version ingested decks; run validation (aoep_shared.validation) and
    only publish corroborated content; route low-confidence items to corrections.

OPERATE
  - Start N workers (one per shard). Monitor HarvestStats
    (fetched/ingested/skipped_license/skipped_dup/errors).
  - It is safe to stop/restart; the queue + dedup sets make it idempotent.

ONLINE CRAWL (OER, schools, public data, search engines)
  Background harvest from the web with a durable SQLite corpus (FTS5 RAG index):

    python3 services/harvester/src/harvester/run.py --crawl --topic "algebra" --daemon

  - Discovery: curated OER portals (OpenStax, OER Commons, data.gov, NASA, …)
    plus search-engine queries (Bing/Google/Brave when API keys are set; offline
    mock seeds when not). Configure keys via AppConfig / env.
  - Fetch: polite HTTP (HARVEST_USER_AGENT, HARVEST_MAX_RPS), license gate first.
  - Generate: full teachable deck (examples, try-it, themes, optional --with-media).
  - Store: ~/.cache/aoep/harvest_corpus.db (override AOEP_HARVEST_CORPUS_DB).
    Tables: sources, chunks (FTS5), courses, persistent queue.
  - Search corpus: --corpus-search "linear equations" [--subject math]
  - One-shot batch: --crawl --topic "machine learning" --max-items 10 --once
  - Seeds file: --crawl --seeds seeds.jsonl  (JSONL SourceSpec rows)
  - Env seeds: HARVEST_SEEDS=/path/to/seeds.jsonl

  Each ingested course lands under output/harvest/courses/<course_id>/ with
  .pptx, .course.json (includes theme: poster, wallpaper, accent), and optional
  media_manifest.json + per-slide audio.

  PPTX IS REQUIRED: every successful ingest writes <title>.pptx + <title>.course.json.
  Install python-pptx before crawling:
    pip install -e 'packages/shared[harvest]'
  Without it, --crawl and --generate exit with install instructions.

  Present a crawled course (Part 3):
    python3 scripts/present_course.py \\
        "output/harvest/courses/<course_id>/<title>.course.json" \\
        --with-media --persona davis --meeting-provider local

  After --crawl, the JSON summary includes a ``packages`` list with course_json,
  pptx, and a ready-to-run ``present`` command when courses exist on disk.

  Crawl until items generate (queue may be large; raise --max-items or drop --once):
    python3 services/harvester/src/harvester/run.py --crawl \\
        --seeds seeds/harassment-training.jsonl \\
        --max-items 20 --once --with-media --out-dir output/harvest

  IMPORTANT: --generate reads YOUR LOCAL FILE. It does not scrape the web.
  The 24/7 web crawl is the default mode when you run run.py with no flags.

  What happens to your file (offline, deterministic — no LLM required):
    1. EXTRACT  pypdf/python-pptx/etc. pull text from the file into sections
                (heading + body pairs).
    2. NORMALIZE Filter TOC junk, dot leaders, and copyright stubs; extract real
                prose from noisy PDF text; merge page splits into learning units.
    3. SLIDE    Build a teachable deck: welcome hook, concept slides (key idea +
                scaffolding bullets + optional fun fact), try-it checkpoints,
                periodic recaps, closing CTA. Speaker notes carry a full script.
                PDF textbooks are chunked into chapter-sized blocks automatically.
    4. SCORE    A pedagogical composition matrix + PCS fingerprint.
    5. TAG      RAG + taxonomy infer subject, tier, price, career_path, core, labels
                (override any field with explicit CLI flags; --no-auto-tags to disable).
    6. OUTPUT   Writes <title>.course.json + <title>.pptx to output/harvest/
                by default. Add --with-media for per-slide narration audio
                (macOS ``say`` / Linux ``espeak``) plus demo-video references
                in media_manifest.json; worked-example slides are always included.

  Minimal local run (from repo root, after ``make install``):
    python3 services/harvester/src/harvester/run.py --generate notes.pdf

  With narration audio + demo-video refs:
    python3 services/harvester/src/harvester/run.py --generate notes.pdf --with-media

  Override a single inferred field when needed:
    python3 services/harvester/src/harvester/run.py --generate notes.pptx \
        --subject chemistry --out-dir output/harvest
    python3 services/harvester/src/harvester/run.py --generate lessons.db \
        --source-type database --query "SELECT topic, body FROM lessons" \
        --heading-column topic --subject biology
    python3 services/harvester/src/harvester/run.py --critique notes.pdf
        -> grade (A-F), issues, and concrete improvement suggestions.

  Generation recipe (aoep_shared.harvest):
    extractors.extract*/extract_file  -> normalized ExtractedDoc (title+sections)
    generate.generate_course          -> slides + numpy CourseComposition + tags
    composition.composition_score()    -> recipe fingerprint (0..999, e.g. 247)
    composition.quality_index()/metrics-> coverage/balance/depth/interactivity
    tagging.CourseTags                 -> free/expensive, LinkedIn job, career
                                          path (nurse), core-fundamental (algebra)

QUANTIFIABLE COMPOSITION + WHY-IT-WORKS LOOP
  - Every class is stored as a numpy node/sub-node matrix. NODES are pedagogical
    categories (introduction, history, example, video, q&a, summary, ...);
    SUB-NODES are subtopics within a category (e.g. "music" under history, or
    "example 2"). See aoep_shared.harvest.NODE_CATEGORIES.
  - composition_score() is the Salareen Pedagogical Composition Score (PCS), a
    custom/patent-pending algorithm that reduces the matrix to one number (the
    "247"/"148" in the spec) that *equates to* which nodes/sub-nodes were used.
    Published formula + data-flow/design diagrams: README "Course Composition
    Score (PCS)". Constants (rho/alpha/beta/mu) are fixed forever:
    QUANT_RESOLUTION, STRUCT_NODE_COEFF, STRUCT_CATEGORY_COEFF,
    DEFAULT_SCORE_MODULUS in composition.py. Key survey happiness on the PCS via
    CompositionOutcomeLedger, then ask:
        ledger.compare(247, 148, subject="chemistry")
    to learn which recipe makes learners happier. quality_index() is our prior.

CONTINUOUS REVIEW / CRITIQUE / OPTIMIZE
  - HarvestCritic grades each generated course and lists issues/suggestions
    (missing intro/summary, low coverage, low interactivity, imbalance).
  - optimize_with_ledger(OptimizationLedger, reports) records each critique pass
    as a revertible checkpoint and promotes it only if average quality does not
    regress - so harvested quality can be tracked and rolled back as the learning
    algorithm evolves.

  Run the suites:
    PYTHONPATH=packages/shared/src python3 -m pytest packages/shared/tests/ -q
    (from services/harvester) python3 -m pytest -q   # entrypoint + CLI smoke

  Optional deps for binary sources (pinned in requirements-dev.txt): pypdf,
  beautifulsoup4, python-pptx, python-docx. Imported lazily, so text/database/
  html generation and the whole crawl loop run without them.

PART 2/3 - TEACH THE GENERATED COURSE, THEN PRESENT IT IN A MEETING
  The harvested course feeds two more stages (offline paths exist for both):
    Part 2 (teach):   aoep_shared.teaching.teach_course -> a LessonPlan. The
                      offline builder reuses the harvester's per-slide narration;
                      set engine="ppt_trainer" to delegate to the external
                      agentic reader (the separate ppt_trainer project) for an
                      LLM-written script + audio/video. Hand-off is a real .pptx:
                      aoep_shared.harvest.export_course_package writes
                      <title>.pptx (narration in the slide SPEAKER NOTES) +
                      <title>.course.json, and `ppt-trainer teach <pptx>` reads
                      the pptx natively.
    Part 3 (present): aoep_shared.meeting presents the lesson live. Providers:
                      Google Meet (Calendar+conferenceData), Zoom (S2S OAuth),
                      Teams (Graph onlineMeetings), and an offline mock. A
                      presenter builds a timed plan and drives slide->speak->
                      advance, emitting a transcript + event log. Real providers
                      activate by env (MEETING_PROVIDER + provider creds) and fall
                      back to the mock when no creds are present.

  One-command end-to-end (from repo root):
    python3 scripts/teach_and_present.py --source notes.pptx --subject chemistry \
        --meeting-provider zoom --teach-engine fallback
  Smoke-test the whole chain (no keys/network/ffmpeg): make validate-pipeline
    (or: python3 scripts/validate_pipeline.py)
