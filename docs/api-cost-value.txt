SALAREEN / AOEP — API COST-TO-VALUE ANALYSIS
============================================
Plain text on purpose (repo no-markdown house rule).

Goal: make the unit economics of every API explicit so we can scale to millions
of learners affordably. The design principle (AGENTS.md) is dual-mode by ENV:
the offline/self-hosted path keeps the whole teaching loop working with NO paid
provider, and paid providers are opt-in upgrades for quality/scale. This doc
maps each capability to its cost driver, the value it unlocks, and how we keep
cost near zero by default.

Companion: docs/api-reference.txt (the APIs), docs/scalability.txt + docs/hosting.txt
(infra economics), docs/payments-and-security.txt (transaction fees).

============================================================================
3. COST DRIVERS RANKED (where the money actually goes at scale)
============================================================================
  1. LLM inference (if cloud LLM enabled) — dominant variable cost. Mitigate by
     self-hosting on reserved GPU, the grounded fallback, caching, RAG to keep
     prompts short, and the optimization ledger to promote cheaper-but-equal
     configs.
  2. Realtime media (LiveKit + bridge egress) for live group classes — scales
     with concurrent minutes. Mitigate by self-hosting LiveKit, capacity caps,
     and recording-on-demand.
  3. Cloud TTS/ASR minutes — only the fallback language subset; on-device speech
     in web/mobile drives this toward $0 for most sessions.
  4. Job-aggregator quotas (JSearch/Adzuna) — bounded by the 15-minute result
     cache and only when keys are configured.
  5. Egress + storage — standard infra; controlled by CDN/cache + retention.

============================================================================
4. VALUE PER DOLLAR (why each paid call is worth it — or skipped)
============================================================================
  - Cloud LLM: highest quality free-form tutoring. Worth it for paid tiers;
    free tier stays on the grounded fallback so we never pay to serve free users
    a feature they didn't pay for.
  - Cloud TTS on the fallback subset: unlocks natural audio for languages XTTS
    can't voice (e.g. Khmer) — directly serves the brand's core audience, so the
    per-char cost buys real reach.
  - JSearch/Adzuna: turns "Careers" from a demo board into real, LinkedIn/Indeed-
    sourced openings -> a concrete acquisition + retention hook. Cache makes the
    per-listing cost negligible.
  - Payments providers: the fee IS the revenue path; localized methods (UPI/PIX/
    etc.) materially raise conversion in target markets (docs/payments.txt).

============================================================================
5. CONTROLLING + MEASURING COST
============================================================================
  - Toggle everything by ENV (config/local.env vs config/cloud.env). Nothing
    paid is on by default; see docs/secrets.txt for the exact keys.
  - Per-service /metrics (Prometheus) exposes request counts + latency; add a
    provider-call counter per metered path to attribute spend.
  - Rate limits (RATE_LIMIT) + the ingress edge limit cap abusive/runaway spend.
  - Caching (ETag/Cache-Control on reads; TTL caches on jobs/translation) cuts
    repeat provider calls.
  - HPA/PDB (infra/k8s/autoscaling.yaml) scale compute with demand so we don't
    pay for idle capacity.

============================================================================
6. RULE OF THUMB FOR ADDING A NEW PROVIDER CALL
============================================================================
  1. Put it behind the provider abstraction with a free/offline fallback.
  2. Gate it on an explicit ENV key; fail-closed (free path or 503), never
     silently bill.
  3. Cache results (with a sane TTL) and add a metric.
  4. Document the cost driver + mitigation in THIS file and the key in
     docs/secrets.txt.
