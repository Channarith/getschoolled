SECRETS & GPU REFERENCE
=======================

All integrations are env/secret-driven. Add values in the Cursor "Secrets" panel
(persisted across runs, injected as env vars) or your deploy env. Nothing below
is committed to the repo. config/local.env and config/cloud.env hold non-secret
defaults; secrets override them by env var.

LEGEND:  [app] main stack  [train] forked GPU agent  [harvest] worker agent

-----------------------------------------------------------------------------
SEARCH / VALIDATION  [app]  (optional - MockSearchProvider is used if unset)
-----------------------------------------------------------------------------
  BING_SEARCH_KEY            Bing Web Search v7 key
  GOOGLE_CSE_KEY             Google Programmable Search key
  GOOGLE_CSE_CX              Google Programmable Search engine id
  BRAVE_SEARCH_KEY           Brave Search API key
  KAGI_API_KEY               Kagi Search API key
  BAIDU_API_KEY              Baidu search key (cn corroboration)
  # Course validation cross-checks claims across whichever engines are configured.

-----------------------------------------------------------------------------
LLM SERVING / ROUTING  [app]
-----------------------------------------------------------------------------
  LLM_BASE_URL               served (vLLM) OpenAI-compatible endpoint = champion model
  LLM_MODEL                  default model name
  LLM_ROUTES                 Track B routing, e.g.
                             "math=adapters/stem,history=adapters/humanities"
  LLM_API_KEY                if the serving endpoint requires auth

-----------------------------------------------------------------------------
MODEL TRAINING  [train]  (see training/RUNBOOK.txt) - needs GPUs
-----------------------------------------------------------------------------
  GPU                        Track B (QLoRA adapters): 1x 24-48GB GPU per domain.
                             Track A (from scratch): multi-node A100/H100 cluster
                             (run the proxy-1b rung first to validate scaling).
  BASE_MODEL                 open-weight base checkpoint (local path or HF id)
  HF_TOKEN                   only if pulling weights from Hugging Face
  CHECKPOINT_DIR             object-store path for checkpoints (Track A)
  TOKENIZER_DIR              object-store path for the trained tokenizer (Track A)
  CORPUS_SOURCES             object-store prefixes of the licensed pretraining corpus
  TARGET_PARAMS / TARGET_TOKENS   final Track A size + token budget

-----------------------------------------------------------------------------
HARVESTER  [harvest]  (see services/harvester/RUNBOOK.txt)
-----------------------------------------------------------------------------
  CURRICULUM_URL             curriculum service base URL (post decks/catalog)
  HARVEST_USER_AGENT         polite contact UA string
  HARVEST_MAX_RPS            global rate limit
  HARVEST_SEEDS              seed list of OER/permissive sources
  OBJECT_STORE_ENDPOINT / OBJECT_STORE_KEY / OBJECT_STORE_SECRET / OBJECT_STORE_BUCKET
                             raw-artifact storage (S3/MinIO; see cloud.env keys)

-----------------------------------------------------------------------------
PAYMENTS  [app]  (optional - SandboxPaymentProvider simulates all methods)
-----------------------------------------------------------------------------
  STRIPE_API_KEY             live/test Stripe key (card, Apple/Google Pay, etc.)

-----------------------------------------------------------------------------
JOBS / CAREERS  [app]  (optional - a curated sample board is used if unset)
-----------------------------------------------------------------------------
  LINKEDIN_API_KEY           LinkedIn jobs API key (live openings)
  JOBS_API_KEY               generic jobs-board/Indeed-style API key
  # Without a key, /jobs serves a curated board representative of LinkedIn/Indeed
  # listings; course<->job skill matching works the same either way.

-----------------------------------------------------------------------------
ADMINISTRATION  [app]
-----------------------------------------------------------------------------
  ADMIN_SECRET               gates administrative feature-flag writes on the
                             memory service (sent as the X-Admin-Secret header).
                             Defaults to "dev-admin-secret" if unset (dev only) -
                             set a strong secret in any shared/prod deployment.
  ENABLE_TEST_ENDPOINTS      enables deterministic automation/E2E test hooks
                             (POST /admin/test/reset|seed). Implicitly on in
                             local/edge mode, off in cloud; set 1/0 to force.
                             These hooks ALSO require ADMIN_SECRET. Keep off in prod.

-----------------------------------------------------------------------------
OBSERVABILITY  [app]  (core is dependency-free; external export is optional)
-----------------------------------------------------------------------------
  Every service exposes (no secret needed):
    GET /metrics            Prometheus exposition (scrape from Prometheus/Grafana)
    GET /telemetry/summary  perf + memory/runtime + error/log counts (JSON)
    GET /telemetry/errors   recent exceptions w/ traceback + request id (RCA)
    GET /telemetry/logs     recent structured events
  Optional external export (only active if the lib is installed in the image):
    SENTRY_DSN                 send errors/traces to Sentry
    SENTRY_TRACES_SAMPLE_RATE  trace sampling (0.0-1.0)
    OTEL_EXPORTER_OTLP_ENDPOINT  OpenTelemetry OTLP collector endpoint

NOTES
  - The whole stack runs fully offline with mocks/sandboxes; secrets only switch
    on real providers. Add only what you need for the path you are exercising.
  - The forked training and harvester agents are SEPARATE long-lived agents with
    their own secrets/compute; the main app does not need the [train] secrets.
