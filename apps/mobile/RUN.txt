AI CLASSROOM - MOBILE APP (Android + iOS)
=========================================
Expo + React Native. Opens into "Drive Mode" (audio-only classes) for learning
on the go.

FIRST-TIME SETUP (macOS — read this first)
==========================================
Two ways to run Salareen on a Mac. Most developers should use Path B (native).

  Path B (RECOMMENDED) — native dev build
    • Compiles ios/ and android/ locally (expo prebuild + CocoaPods/Gradle).
    • No Expo Go install; correct launcher icon; fewer moving parts.
    • First run: 10–20 min with little output — this is normal.
    • Daily (verbose):  pnpm run ios:debug   /  pnpm run android:debug
    • npm equivalent:    npm run ios:debug     /  npm run android:debug

  Path A — Expo Go (optional; higher friction)
    • Separate Expo Go app must be installed on each simulator (network once).
    • Good for quick JS-only tweaks; painful for first-time setup.
    • Setup wizard:  bash scripts/mobile-setup.sh   or   pnpm run setup
    • Daily (verbose):  pnpm run launch:ios:debug

  NEVER on Mac:  expo start ...   or   pnpm exec expo ...
  Always use bash wrappers (12 GB Node heap for Metro).

MACOS REQUIREMENTS (install once)
  1. Xcode from App Store → open once → finish setup.
  2. Accept license:
       sudo xcodebuild -license accept
  3. iOS Simulator runtime (REQUIRED after every major Xcode update):
       Xcode → Settings → Platforms → download latest iOS Simulator
     If simulators fail to boot: reboot the Mac, then:
       open -a Simulator
       xcrun simctl list devices available | head
  4. Node 18+ and pnpm:
       npm i -g pnpm
  5. CocoaPods (iOS native builds):
       sudo gem install cocoapods
  6. Android Studio + AVD (Android only):
       Device Manager → Create Virtual Device (e.g. Pixel 7, API 34+)
       Add to ~/.zshrc:
         export ANDROID_HOME=$HOME/Library/Android/sdk
         export ANDROID_SDK_ROOT=$ANDROID_HOME
         export PATH=$ANDROID_HOME/emulator:$ANDROID_HOME/platform-tools:$PATH

INSTALL (one-time — prefer mobile-setup.sh)
  cd apps/mobile
  bash scripts/mobile-setup.sh     # deps + doctor + Expo Go (recommended)
  pnpm run setup                   # same via package.json
  # Legacy manual install:
  pnpm install
  # Or, from the repo root:
  make mobile-install
  # NOTE: apps/mobile/.npmrc sets `node-linker=hoisted` and

  # `pnpm install` if you switch package managers.

NOTHING HAPPENS / EMULATOR DOES NOT OPEN (macOS)
  Symptom: launch script sits at "Starting Metro Bundler" or "rebuilding cache"
  for several minutes with no Simulator window.

  This is NORMAL on first run (2–5 min). Metro builds ~735 JS modules before
  Expo Go opens. Do NOT use --clear every time (removed from default launch).

  Timeline:
    1–3 min   Metro + Watchman (may pause with no new lines)
    3–5 min   First bundle — look for "iOS Bundled" / "Opening on iOS"
    Then      Expo Go should appear in Simulator with Salareen

  Faster relaunch (cache kept):
    bash scripts/mobile-launch-ios.sh

  Only if Metro acts broken:
    bash scripts/mobile-launch-ios.sh --fresh

  If 5+ min with zero progress:
    bash scripts/mobile-launch-ios.sh --debug
    # or open Simulator first: open -a Simulator
    # then in two steps:
    bash scripts/mobile-expo.sh start
    # press i in that terminal when Metro is ready

  Stuck at "Starting Metro Bundler" / "Fetching bundled native modules":
    Cause: stale Metro on port 8081 or expo.dev network hang (when EXPO_OFFLINE unset).
    Fix:
      bash scripts/mobile-metro-cleanup.sh
      bash scripts/mobile-launch-ios.sh
    Launch scripts: debug ON by default, CI=false, EXPO_OFFLINE=1 (local bundled modules).
    First run without Expo Go: bash scripts/mobile-install-expo-go-ios.sh (network once).

  "Expo Go is not installed … while running in offline mode":
    Run: bash scripts/mobile-install-expo-go-ios.sh
    Then: bash scripts/mobile-launch-ios.sh

  Symptom: you run pnpm ios or pnpm start and the terminal returns or sits
  with no Simulator window.

  1. Wrong directory — commands MUST run from apps/mobile, not the repo root.
       cd apps/mobile && pnpm run doctor

  2. Wrong command — pnpm ios / npm run ios is a NATIVE compile (Path B), not Expo Go.
       Expo Go:  pnpm run launch:ios:debug
       Native:   pnpm run ios:debug

  3. Open the Simulator manually before launching:
       open -a Simulator
       pnpm run dev:ios

  4. See verbose Expo logs:
       EXPO_DEBUG=1 pnpm run dev:ios

  5. Expo cannot reach expo.dev (corporate VPN / offline):
       pnpm run dev:ios:offline

  6. Xcode license / simulators missing or won't boot:
       sudo xcodebuild -license accept
       open -a Xcode   # Settings -> Platforms -> install iOS Simulator runtime
       reboot Mac if simctl still fails after installing runtime

  7. Clean reinstall:
       rm -rf node_modules .expo && pnpm install && pnpm run dev:ios -- --clear

  8. Native build (pnpm ios) stuck with no output — it may still be compiling.
       Watch Activity Monitor for node/xcodebuild, or run in another terminal:
       EXPO_DEBUG=1 pnpm ios

macOS — iOS SIMULATOR SETUP (one-time)
  1. Install Xcode from the App Store; open Xcode once and finish setup.
  2. Accept the license if prompted:
       sudo xcodebuild -license accept
  3. Verify the simulator toolchain:
       xcode-select -p
       xcrun simctl list devices available | head
  4. From apps/mobile:
       pnpm install
       pnpm run start:ios
     The first launch downloads Expo Go into the simulator; allow a few minutes.

macOS — ANDROID EMULATOR SETUP (one-time)
  1. Install Android Studio.
  2. Device Manager -> Create Virtual Device (e.g. Pixel 7, API 34).
  3. Add to ~/.zshrc (or ~/.bash_profile):
       export ANDROID_HOME=$HOME/Library/Android/sdk
       export ANDROID_SDK_ROOT=$ANDROID_HOME
       export PATH=$ANDROID_HOME/emulator:$ANDROID_HOME/platform-tools:$PATH
     Reload the shell:  source ~/.zshrc
  4. List and boot an AVD (or start it from Android Studio):
       emulator -list-avds
       emulator -avd Pixel_7_API_34 &
  5. From apps/mobile (with the emulator running):
       pnpm install
       pnpm run start:android

TROUBLESHOOTING — EMULATOR / SIMULATOR WON'T LAUNCH
  "pnpm ios" or "pnpm android" fails on first run
    - These run  expo run:ios / expo run:android  (native compile), NOT Expo Go.
      Try  pnpm run start:ios  or  pnpm run start:android  first.
    - Native first run needs network for CocoaPods (iOS) and Gradle/Maven
      (Android). If prebuild is required:  pnpm run prebuild  then retry.

  iOS: "xcodebuild" / "Unable to boot simulator" / no simulators listed
    - Open Xcode -> Settings -> Platforms; install an iOS simulator runtime.
    - Run:  sudo xcodebuild -license accept
    - Pick a device:  pnpm run start:ios -- --simulator "iPhone 16"

  Android: "SDK location not found" / "emulator: command not found"
    - Set ANDROID_HOME (see above) and ensure  emulator  is on PATH.
    - Boot the AVD BEFORE  pnpm run start:android  (Android Studio -> Play on
      the virtual device).

  Metro bundler errors / "Unable to resolve module"
    - From apps/mobile:  rm -rf node_modules && pnpm install
    - Clear Metro cache:  pnpm start -- --clear

  App opens but home rails are empty / network errors
    - Start the curriculum API on your Mac (port 8005 — see BACKEND URL below).
    - Simulator: localhost works. Physical device: use your Mac's LAN IP in
      app.json -> expo.extra.curriculumUrl (not localhost).

  pnpm / Metro symlink issues
    - apps/mobile/.npmrc forces hoisted node_modules; always use pnpm install
      inside apps/mobile (not npm install at the repo root for mobile).

TYPECHECK
  From the monorepo root:
    cd apps/mobile
  If your shell prompt already ends in "mobile" and app.json is here, you are
  in the right place — do NOT run "cd apps/mobile" again (that path is wrong).

  JavaScript heap out of memory (typecheck, export, dev, launch — Mac Node 22)
  ---------------------------------------------------------------------------
  Symptom: FATAL ERROR near ~4040 MB heap, AfterScanDir in stack trace.

  Cause: Node 22 defaults to ~4 GB. Metro runs in a subprocess that ignores
  NODE_OPTIONS unless METRO_NODE_OPTIONS is also set. pnpm exec expo adds more
  heap pressure scanning hoisted node_modules.

  Fix (git pull origin main) — all expo/dev commands use bash wrappers with 12 GB:
    scripts/mobile-env.sh       NODE_OPTIONS + METRO_NODE_OPTIONS
    scripts/mobile-expo.sh      expo without pnpm exec
    scripts/mobile-launch-ios.sh / mobile-launch-android.sh

  RUN ON SIMULATOR / EMULATOR (use these, not raw expo):
    bash scripts/mobile-launch-ios.sh
    bash scripts/mobile-launch-android.sh
    bash scripts/mobile-expo.sh start --ios --clear
    bash scripts/mobile-expo.sh start --android --clear

  Or via pnpm (now wraps bash internally):
    pnpm run launch:ios
    pnpm run dev:android

  NEVER on Mac:  expo start ...   or   pnpm exec expo ...

  If still OOM, raise heap in your shell before any command:
    export NODE_OPTIONS=--max-old-space-size=12288
    export METRO_NODE_OPTIONS=--max-old-space-size=12288

  If export OOMs (often on second platform after ios succeeds):
    export scripts set NODE_OPTIONS + METRO_NODE_OPTIONS=8192 and run each
    platform in a separate Node process. Prefer bash directly on Mac:
      bash scripts/mobile-export.sh ios
      bash scripts/mobile-export.sh android
    Or: pnpm run export:ios  then  pnpm run export:android  (one at a time)

  Verbose export:
    VERBOSE=1 bash scripts/mobile-export.sh ios

NATIVE PROJECT FILES (offline / custom CI)
  pnpm run prebuild       # writes apps/mobile/{android,ios}/ (gitignored)
  # Run this if you need Gradle / Xcode projects to build locally without EAS.
  pnpm run native:prebuild:android  # Android-only clean prebuild + Gradle patch
  pnpm run native:build:android     # local Android debug APK build
  The Android prebuild command patches the generated Gradle wrapper to download
  Gradle 8.8 from the official Gradle GitHub distribution release. Override with
  GRADLE_DISTRIBUTION_URL if your build network requires a different mirror. It
  also disables React Native's Foojay Java-toolchain resolver plugin for the
  local build because Java is supplied by CI/local setup and some restricted
  networks block plugins.gradle.org.

BACKEND URL
  The app reads API URLs from app.json -> expo.extra (curriculumUrl, identityUrl,
  memoryUrl). src/config.ts auto-maps Android emulator to 10.0.2.2 (host Mac).

  iOS Simulator: localhost works (default in app.json).
  Android Emulator: leave defaults — config uses 10.0.2.2 automatically.
  Physical device: set all three URLs to your Mac's LAN IP or deployed host.

  Start local backends (from repo root, separate terminals):
    cd services/curriculum && DEPLOY_MODE=local PYTHONPATH=src \
      uvicorn curriculum.main:app --port 8005
    cd services/memory && DEPLOY_MODE=local PYTHONPATH=src \
      uvicorn memory.main:app --port 8004
    cd services/identity && DEPLOY_MODE=local PYTHONPATH=src \
      uvicorn identity.main:app --port 8008

LANGUAGES (i18n)
  The UI is translated into 13 languages out of the box: English, Spanish,
  French, German, Italian, Portuguese, Russian, Arabic, Hindi, Chinese
  (Mandarin), Japanese, Korean, Vietnamese. The other 13 supported languages
  (Dutch, Polish, Ukrainian, Turkish, Hebrew, Bengali, Urdu, Persian, Thai,
  Indonesian, Swahili, Greek, Czech) appear in the picker and inherit
  English until they're translated.
  - Initial locale: AsyncStorage user pick > device locale (expo-localization)
    > English fallback.
  - Switch language: Settings -> LANGUAGE section -> tap a flag chip.
  - RTL: Arabic / Hebrew / Urdu / Persian flip layout via `direction: rtl`.
  - Backend: GET /notifications/feed accepts `locale=es|fr|de|...` and renders
    titles/bodies in 13 languages. Unknown locales fall back to English.
    Source: aoep_shared/notifications.py.

NETFLIX-STYLE FEATURES (all on the audio catalog)
  Push / local notifications via expo-notifications:
    * Daily class reminder (user-configurable hour, OS-scheduled, repeating)
    * "New class" + "Picked for you" banners (staggered local alerts)
    * Completion alert when a class finishes
    * Tappable - deep-links into Drive Mode for the right course
  In-app alerts: top banner that surfaces foreground notifications with CTA.
  Inbox: GET /notifications/feed renders a personalized list (new / continue /
    recommended / streak / daily); read-state stored in AsyncStorage; unread
    count appears as a badge on the Alerts tab.
  Continue Listening: every play tick records progress; the Home screen rail
    resumes from the saved segment.
  My List: star icon on every card / row toggles a saved-courses list.
  Streak: each completed class extends a daily streak (used by the inbox feed).
  Browse rails: Continue, My List, New This Week, Picked for You, Trending,
    and a Categories tile grid.
