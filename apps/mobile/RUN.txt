AI CLASSROOM - MOBILE APP (Android + iOS)
=========================================
Expo + React Native. Opens into "Drive Mode" (audio-only classes) for learning
on the go.

PREREQUISITES
  - Node 18+ and pnpm (`npm i -g pnpm`). The repo's CI / Cursor Cloud env
    installs pnpm already.
  - Expo Go on your phone, an Android emulator (Android Studio), or the iOS
    Simulator (macOS / Xcode) for live dev.

INSTALL (one-time)
  cd apps/mobile
  pnpm install
  # Or, from the repo root:
  make mobile-install
  # NOTE: apps/mobile/.npmrc sets `node-linker=hoisted` because Metro/React
  # Native cannot resolve through pnpm's strict symlink layout. Re-run
  # `pnpm install` if you switch package managers.

DEV LOOP
  cd apps/mobile
  pnpm start             # then scan the Expo Go QR, or:
  pnpm android           # Android emulator/device
  pnpm ios               # iOS simulator (macOS)

TYPECHECK
  pnpm typecheck         # or: make mobile-typecheck

BUILD (production JS bundles - iOS + Android)
  pnpm run export        # writes apps/mobile/dist/
  # or: make mobile-build
  # Produces:
  #   dist/_expo/static/js/ios/index-<sha>.js     (~970 kB minified)
  #   dist/_expo/static/js/android/index-<sha>.js (~978 kB minified)
  #   dist/metadata.json
  # These are the JS bundles that EAS Build / OTA updates ship.
  # `pnpm run export:ios` / `pnpm run export:android` build just one platform.

NATIVE BINARIES (.apk / .aab / .ipa)
  Use EAS Build - it provisions Android SDK + Xcode in the cloud, no local
  Android Studio / Mac needed.
    npm i -g eas-cli            # one-time
    eas login                   # one-time, requires an Expo account
    cd apps/mobile
    pnpm run build:android      # production AAB
    pnpm run build:ios          # production IPA (needs Apple Developer team)
    pnpm run build:preview:android   # internal-distribution APK
  Build profiles live in apps/mobile/eas.json. App identifiers:
    ios.bundleIdentifier: com.aiclassroom.app
    android.package:      com.aiclassroom.app

NATIVE PROJECT FILES (offline / custom CI)
  pnpm run prebuild       # writes apps/mobile/{android,ios}/ (gitignored)
  # Run this if you need Gradle / Xcode projects to build locally without EAS.
  # NOTE: prebuild fetches a template tarball from npm; it will hang on a
  # locked-down network. Use EAS Build instead in that case.

BACKEND URL
  The app reads the curriculum API from app.json -> expo.extra.curriculumUrl.
  On a physical device, "localhost" points at the phone, not your computer, so:
    - set curriculumUrl to your machine's LAN IP (e.g. http://192.168.1.20:8005), or
    - point it at your deployed backend.
  Make sure the curriculum service is running:
    cd services/curriculum && DEPLOY_MODE=local PYTHONPATH=src \
      uvicorn curriculum.main:app --port 8005

WHAT'S HERE
  App.tsx                       root (list <-> drive player)
  src/api.ts                    fetches /audio/courses from curriculum
  src/screens/AudioCoursesScreen.tsx   browse audio classes (search + categories)
  src/screens/DriveModeScreen.tsx      hands-free player (expo-speech narration)
  eas.json                      EAS Build profiles (development/preview/production)
  .npmrc                        forces hoisted node_modules for Metro

ROADMAP
  - Auth (reuse identity /auth + token) and sync points/streaks.
  - expo-av for real audio files / background playback + lock-screen controls.
  - Live class, browse, and language-learning screens (reuse the same APIs).
  - Offline download of audio lessons for tunnels / no-signal drives.
