[0.3.5] - 2026-06-18
- catalog API (P5): curriculum endpoints for the catalog - POST/GET/DELETE
  /courses and /programs, GET /catalog (tree), and POST /programs/{id}/plan which
  orders a program's courses and gates each by its prerequisite-mastery rule
  (returns next_course + per-course unlocked/reason). CatalogStore wired at
  CATALOG_PATH. Offline-tested.

[0.3.4] - 2026-06-18
- catalog (P4): course catalog + dynamic training programs. curriculum/catalog.py
  - Module/Course/Program models + CatalogStore (CRUD + JSON persistence,
  autosave) where Module references CMS decks/scenes and Program carries adaptive
  rules (e.g. prereq mastery). db/migrations/0003_catalog.sql schema-of-record
  (courses/modules/programs/program_courses/enrollments). Offline-tested.

[0.3.3] - 2026-06-18
- validation (P2/P3): aoep_shared/validation.py - extract_claims, validate_claim,
  and validate_course corroborate course claims across the configured search
  engines via lexical-overlap (supported/unverified/contradicted + confidence +
  citations). Curriculum endpoints POST /validate/claim and POST
  /decks/{id}/validate (report with flagged claims). Offline-tested with the mock
  engine; real engines used when keys are configured.

[0.3.2] - 2026-06-18
- search/validation (P1): add a pluggable SearchProvider abstraction for course
  validation - SearchResult + SearchProvider in providers/base.py; providers/
  search.py with key-gated adapters (Bing, Google CSE, Brave, Kagi, Baidu) plus
  a deterministic MockSearchProvider; per-engine API-key config; factory
  .search_engines() returns every configured engine (mock when none). Offline-
  tested; real engines activate when their key/secret is set.

[0.3.1] - 2026-06-18
- fix(changelog): repair changelog rolling. build_release.py only recognized a
  dated "[unreleased] - <date>" header, so the bare "[unreleased]" header was
  never rolled into a release: real notes piled up under [unreleased] while the
  bot appended empty "[0.2.x] - release build (no itemized changes)" sections.
  The regex now matches both forms; the corrupted cascade was collapsed and the
  accumulated notes consolidated into [0.3.0].
- training: add offline (Linux/Ubuntu) training code. training/export.py builds
  JSONL from class sessions/feedback; training/run_finetune.py runs QLoRA/LoRA
  fine-tuning (transformers+peft+trl, 4-bit on CUDA, CPU/LoRA fallback, fully
  offline via HF_HUB_OFFLINE) with a torch-free --check dry run;
  training/evaluate.py scores a checkpoint; training/train.sh is a one-command
  Ubuntu runner. Fairness guardrail (no protected attributes) still enforced.

[0.3.0] - 2026-06-17
- tutor: culture-aware slang & idiom understanding (aoep_shared.slang). A
  data-driven lexicon keyed by language + region (US/UK English, Mexican Spanish,
  French, Khmer, ...) with a normalizer that detects idioms/slang and expands
  them to plain meaning (longest-match, no overlap). The Tutor normalizes a
  question before RAG/LLM so "it's a piece of cake" is understood as "very easy",
  surfaces recognized expressions in the answer (understood[]), and a new
  POST /api/slang/normalize endpoint exposes it.
- perception: machine-vision engagement + cross-session student memory.
  Cross-session memory: FaceGallery persists embeddings (save_json/load_json);
  VisionProvider loads/saves at VISION_GALLERY_PATH so enrolled students are
  remembered across sessions/restarts. Engagement (aoep_shared.vision.engagement):
  real gaze/attention + a coarse expression proxy (neutral/smiling/surprised)
  from the detector's 5 landmarks; analyze_image returns attention/gaze_frontal/
  expression. GestureRecognizer interface scaffolded. Engagement feeds the
  phase-4 adaptive policy.
- web: audience-tailored landing templates. Data-driven theme registry
  (apps/web/app/lib/themes.ts) + a Template switcher: Corporate, Khmer Heritage
  (Angkor Wat), Culinary, Kids & Gamified, Academic. Selection persists and is
  shareable via ?theme=. Themed art under apps/web/public/themes/.
- format: AOEPLX layered live-teaching scene format (aoep_shared.scene +
  docs/aoeplx-format.txt). Ordered layers (video/audio/image/whiteboard/text/
  note/quiz/poll) with transforms, time ranges, anchored attachments; live
  SceneDeltas; extract_region lifts a bbox/time slice into a multilayer object
  (image/clip); tamper-evident SHA-256 + HMAC container. Curriculum exposes
  /scenes (+delta/extract/sign) and /scenes/verify.
- scraper: content ingestion -> presentation decks (curriculum.ingest). Extracts
  from textbook PDFs (pypdf), web/HTML (BeautifulSoup), and video/YouTube
  transcripts; generates genre-tagged decks (lecture/hands_on/tutorial/video/
  article) stored in the CMS. Endpoints POST /ingest/pdf,/html,/url,/transcript,
  /youtube.
- training: scaffold the education-LLM training subsystem (training/). Tested
  data pipeline (pipeline.dataset/feedback) turning class transcripts + audience
  context + feedback into reward-weighted JSONL; finetune config. FAIRNESS
  GUARDRAIL: protected attributes (race, ethnicity) excluded from conditioning.
- version: single source-of-truth version (aoep_shared.version.get_version) in
  every service /health + OpenAPI and the web nav; Dockerfiles COPY VERSION;
  build_release syncs apps/web version.
- ci: Release workflow (.github/workflows/release.yml) cuts a GitHub Release
  (tag vX.Y.Z + CHANGELOG notes) when VERSION advances; include training/tests.
- docs: README build-status table updated to reflect implemented phases.
- phase6: curriculum CMS - deck authoring/import/management (SlideSpec/Deck +
  DeckStore + plain-text import) with /decks endpoints.
- phase2: multilingual delivery routing (aoep_shared.translation) across 26
  languages; speech /delivery/plan + /translate.
- phases7-9: media bridge scaffold (aoep_shared.bridges) - MediaBridge interface
  + Zoom/Teams(.NET)/Meet registry + credential-readiness checks.
- phase10: latency budget + recorder (aoep_shared.observability).
- refactor: consolidate to a single stack (remove duplicate eduplatform_shared +
  services/*/app/ layouts; rename dist to aoep-shared; port the web teaching loop
  into the aoep orchestrator with an offline grounded fallback).
- phase4: adaptive learning policy (aoep_shared.adaptive) -> pacing/difficulty/
  re-teach; /director/plan; memory /behavior + /learner.
- phase5: assessment (aoep_shared.assessment) - MCQ generation, difficulty-
  weighted grading, polls; /assessment/quiz + /assessment/grade.
- billing: multiple payment methods (card, Apple/Google Pay, Cash App, PayPal,
  Venmo, Zelle) behind PaymentProvider; /payment-methods + method-aware /checkout.
- perception: real self-hosted face recognition (OpenCV YuNet + SFace, CPU),
  consent-gated enroll/identify; /enroll, /identify, /gallery.
