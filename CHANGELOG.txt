CHANGELOG - Agentic Online Education Platform
=============================================
Format: newest first. No markdown per project convention.

[unreleased]
- web: audience-tailored landing templates. The front page is now re-skinnable
  via a data-driven theme registry (apps/web/app/lib/themes.ts) + a Template
  switcher: Corporate, Khmer Heritage (Angkor Wat), Culinary, Kids & Gamified,
  Academic. Each template sets palette, fonts, hero copy, background art, and a
  gamified flag; selection persists (localStorage) and is shareable via ?theme=.
  Themed background art added under apps/web/public/themes/. Adding a new
  audience is a config-only change.
- format: add AOEPLX, our own layered live-teaching scene format
  (aoep_shared.scene + docs/aoeplx-format.txt). Scenes are lightweight
  asset-by-reference manifests of ordered layers (video/audio/image/whiteboard/
  text/note/quiz/poll) with normalized transforms, time ranges, and anchored
  attachments (attach a note/audio/video to a specific drawing). Live broadcast
  via small SceneDeltas (add/update/remove layer, append stroke). extract_region
  lifts a bbox/time slice into a new multilayer object (instant=image,
  range=clip) - e.g. pause a video, draw, and lift the drawing out. Tamper-
  evident container: SHA-256 content hash + HMAC-SHA256 signature
  (sign/verify/serialize/deserialize), versioned. Curriculum service exposes
  POST /scenes, GET /scenes/{id}, /scenes/{id}/delta, /scenes/{id}/extract,
  /scenes/{id}/sign, /scenes/verify. The browser canvas renderer + realtime
  delta transport are the next (frontend/runtime) step.
- scraper: content ingestion -> presentation decks (curriculum.ingest). Extracts
  from textbook PDFs (pypdf), web articles/HTML (BeautifulSoup), and video/YouTube
  transcripts; generates slide decks tagged by class genre (lecture, hands_on,
  tutorial, video, article) and stores them in the CMS. New endpoints:
  POST /ingest/pdf, /ingest/html, /ingest/url, /ingest/transcript, /ingest/youtube
  (url/youtube are runtime/network; youtube dep is optional + import-guarded).
- training: scaffold the education-LLM training subsystem (training/). Real,
  tested data pipeline (pipeline.dataset/feedback): turns class transcripts +
  audience context + user feedback into reward-weighted JSONL training examples;
  finetune config (QLoRA, continuous schedule). FAIRNESS GUARDRAIL: protected
  attributes (race, ethnicity) are excluded from model conditioning by design
  (allowlisted pedagogical features only); enforced + tested. Actual 24/7
  fine-tuning runs on a separate GPU cloud agent (see training/README.txt).
- ci: include training/tests in the test matrix.
- version: surface a single source-of-truth version in the running app.
  aoep_shared.version.get_version() resolves AOEP_VERSION env -> repo VERSION
  file -> package metadata; every FastAPI service reports it in /health (and the
  OpenAPI version), and the web nav shows it. Dockerfiles COPY VERSION into the
  image. scripts/build_release.py now also syncs apps/web (version.ts +
  package.json) and the auto-push-build workflow commits those.
- ci: add Release workflow (.github/workflows/release.yml) that cuts a GitHub
  Release (tag vX.Y.Z + notes from CHANGELOG) when VERSION advances, via
  workflow_run after Auto Push Build; idempotent (skips if the tag exists).
- docs: update README build-status table and descriptions to reflect reality -
  phases 0/3/4/5/6/6b implemented & tested; 1/2/10 partial (offline logic done,
  infra pending); 7-9 scaffolded behind interfaces. Refresh repo-layout,
  tech-stack, and testing notes (120+ tests).
- phase6: curriculum CMS. Deck authoring/import/management - SlideSpec/Deck
  models + DeckStore CRUD + plain-text import parser (curriculum.decks), with
  endpoints POST /decks, POST /decks/import, GET /decks, GET /decks/{id},
  GET /decks/{id}/presentation, DELETE /decks/{id}.
- phase2: multilingual delivery routing (aoep_shared.translation). Per-student
  plan (translate? pair supported? TTS engine native-vs-fallback) across the 26
  languages; speech endpoints POST /delivery/plan and POST /translate (validates
  the pair, then needs the NLLB model).
- phases7-9: media bridge scaffold (aoep_shared.bridges) - MediaBridge interface
  + Zoom/Teams(.NET)/Meet capability registry + credential-readiness checks;
  concrete connect raises BridgeUnavailable until SDKs/creds are configured.
- phase10: latency budget + recorder (aoep_shared.observability) for the
  speech-in -> think -> speech-out loop (1100ms target). Tests: 120 passing.
- refactor: consolidate to a single stack. Remove the duplicate
  eduplatform_shared package and all services/*/app/ layouts; the canonical
  stack is aoep_shared (dist renamed eduplatform-shared -> aoep-shared) with the
  services/*/src layout. Port the web-facing teaching loop (GET /api/lessons,
  POST /api/sessions, /api/sessions/{id}/advance, /api/sessions/{id}/ask) into
  the aoep orchestrator (RAG via aoep_shared.rag + LLM provider, with a
  deterministic offline fallback so the demo works without a model server), so
  apps/web is unchanged. Update Dockerfiles to the src layout, requirements-dev,
  and AGENTS.md. Full suite: 101 tests passing.
- phase4: adaptive learning policy (aoep_shared.adaptive). Turns per-student
  learning-behavior signals (mastery, quiz accuracy, response latency, attention
  trend, question rate) into a pacing/difficulty/re-teach plan; solo personalizes
  aggressively while group caps extremes. Wired into the Director (plan()) and a
  new orchestrator POST /director/plan. Memory service records behavior events
  (POST /behavior) and aggregates them (GET /learner/{id}/{topic}).
- phase5: assessment (aoep_shared.assessment). Definition-check MCQ generation
  from curriculum passages, difficulty-weighted grading -> mastery target, and
  polls (vote/tally/leader). New orchestrator POST /assessment/quiz and
  /assessment/grade. Extensive tests (orchestrator 22, memory 11 passing).
- billing: add multiple consumer payment methods behind PaymentProvider -
  card, Apple Pay, Google Pay, Cash App Pay (Stripe), PayPal + Venmo (PayPal),
  and Zelle (manual bank transfer). New aoep_shared.payments registry maps each
  method to its processor. Sandbox simulates every method (offline-testable);
  Stripe advertises only what it can process and rejects others. Billing API
  adds GET /payment-methods and a method-aware POST /checkout (Zelle returns
  transfer instructions). Extensive tests (25 passing).
- perception: implement real, self-hosted face recognition (OpenCV YuNet
  detection + SFace 128-d embeddings, CPU). New aoep_shared.vision package
  (FaceRecognitionEngine, FaceGallery, runtime model fetch; weights kept out of
  repo). VisionProvider now enrolls + matches (1:N) with the SFace cosine
  threshold, restricted to the consented set (unconsented = anonymous).
- perception API: POST /enroll/{student_id}, POST /identify (consent-gated),
  GET /gallery; consent-check remains a pure policy endpoint.
- tests: extensive face-recognition suite on a real multi-image dataset -
  detection, same-vs-different separation, 1:N accuracy, open-set rejection,
  multi-face frames, consent gating, plus pure gallery unit tests (18 passing).

[0.2.0] - 2026-06-17
- phase0: implement foundations end to end (monorepo, dual-mode, CI).
- packages/shared: AppConfig dual-mode loader (DEPLOY_MODE + per-component
  *_MODE overrides), provider interfaces (LLM/Speech/Vision/Media/ObjectStore/
  Payment) with local+cloud impls, config-driven ProviderFactory, cross-service
  pydantic schemas, 26-language coverage + TTS fallback policy, entitlements
  can_start gate, dependency-free RAG skeleton, shared FastAPI /health + CORS.
- services: orchestrator (Teaching Director state machine + class/join/token),
  speech (languages + TTS engine routing), perception (consent-gated identity),
  memory (profiles/consent/mastery), curriculum (RAG search), billing
  (plans/entitlements/checkout). Each exposes /health and has a Dockerfile.
- apps/agent-runtime: provider-driven TeachingBrain + LiveKit worker entrypoint.
- apps/web: Next.js (app router) dashboard, live-class join, consent UI.
- config/local.env + config/cloud.env (single dual-mode contract).
- db/migrations/0001_init.sql: identity, classes, consent/compliance (FERPA/
  GDPR/BIPA), face embeddings, mastery, curriculum vectors, billing tables
  (pgvector + pgcrypto).
- infra/compose: full local stack; infra/k8s: namespace/config/services/web +
  GPU vLLM serving; models/serving + models/finetune configs.
- bridges/{zoom,teams,meet}: phase7-9 scaffolds.
- CI: GitHub Actions (python tests, web typecheck/build, compose + k8s render);
  Makefile.
- verified: 49 Python tests pass; web typechecks + builds; live local-mode
  endpoints and the local/cloud provider switch exercised over HTTP.
